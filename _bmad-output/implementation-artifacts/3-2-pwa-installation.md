# Story 3.2: PWA Installation

Status: ready-for-dev

## Story

As a player,
I want to install ch3ss to my phone's home screen,
So that it feels like a native app and I can launch it with one tap.

## Acceptance Criteria

1. A Web App Manifest is served with: app name "ch3ss", icons (192x192, 512x512), theme color, standalone display mode (FR20)
2. The browser allows adding ch3ss to the home screen (Chrome install prompt, Safari Add to Home Screen)
3. Once installed, the app launches in standalone mode (no browser chrome — no URL bar, no browser tabs)
4. The app icon appears on the device home screen
5. The manifest theme_color and background_color match the dark (sable) theme

## Dev Notes

### Architecture Compliance

- **Manifest generated by vite-plugin-pwa** — no manual `manifest.json` file needed. The plugin generates it from the config.
- **Icons in `public/icons/`** — matches architecture spec directory structure
- **Standalone display mode** — the app runs fullscreen without browser chrome, like a native app

### Icon Requirements

The architecture spec names the icons `icon-192.png` and `icon-512.png` in `public/icons/`. For a placeholder during dev, use simple chess-themed graphics. These can be refined later.

Android requires `maskable` purpose on the 512px icon for adaptive icons. Ensure the important visual content stays within the inner 80% (safe zone). Best practice: set `purpose: 'any maskable'` on the large icon.

### Safari-Specific PWA Behavior

Safari does NOT fully support the Web App Manifest `icons` field for home screen icons. Required workarounds in `index.html`:
- `<link rel="apple-touch-icon">` — provides the home screen icon
- `<meta name="apple-mobile-web-app-capable" content="yes">` — enables standalone mode
- `<meta name="apple-mobile-web-app-status-bar-style">` — controls status bar appearance

### Theme Color Alignment

The `theme_color` in the manifest MUST match the dark sable theme. Check `src/index.css` from Story 1.1 for the exact background color used. Common dark sable values: `#1a1a2e`, `#16213e`, `#0f0f23`. Use whatever was established in Story 1.1.

If the user has a light theme toggle (Story 2.2), the meta tag `theme-color` is static — it won't change dynamically. This is acceptable for MVP.

### Dependencies

- **Story 3.1 (Service Worker)**: Must be implemented first — PWA installability requires an active Service Worker. The VitePWA plugin config in `vite.config.ts` is shared between Story 3.1 and 3.2.
- **Story 1.1 (Project Foundation)**: `index.html` and `public/` directory must exist.

### File Structure Impact

- **Modifies**: `vite.config.ts` (add manifest config to existing VitePWA), `index.html` (add meta tags)
- **Creates**: `public/icons/icon-192.png`, `public/icons/icon-512.png`
- **No new source files** — PWA installation is config + assets only

### Project Structure Notes

- `public/icons/` directory specified in architecture spec
- Manifest is auto-generated by vite-plugin-pwa — do NOT create a manual `public/manifest.json`
- vite-plugin-pwa generates the manifest at `dist/manifest.webmanifest` during build
